For me, coding ~~is~~ *was* (rip) an art, both a means and an end. Artisanal, small batch, locally sourced, single origin, organic software. 

I started hacking C audio in 1997, and have weathered many system changes, limitations, transitions and deprecations. If you're reading this here on github, you know things have suddenly changed. This is different. Sometimes this feels more like sculpting software than writing it. In composition terms, it's analogous to writing a score by pencil vs composing by ear in a DAW. It's a different process, both create music, but the manner and speed is completely different. I don't feel that speed is always a boon - there is value in slowness that exists outside of this topic. In this case, the result is better, cleaner, more pragmatic software - so what is there to resist about that?

Currently, I'm working independently to create media systems that inspire me to craft, but also interest to me to utilize as a hand tool for music composition. I'm pushing hard to finish the first of these products this spring (2026). Many components will be open sourced here. I don't know how much meaning sharing software has any longer, but community will still matter in our world - no matter how things are assembled/disassembled from now on.
